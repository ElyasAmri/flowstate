Smart Government & Citizen Services · powered by Fanar

One-line pitch. Team: Osama, Elyas.

The Problem

  • Routine applications are delayed, inconsistent, and expensive when every case waits on a person.
  • Most cases are automatable; only a few are true exceptions.
  • We need accountability: replayable and appealable decisions.
  • Arabic / Gulf government context matters — dialect, local procedures.
25% of scoring: importance, impact, value prop.

Solution Architecture

  • Discovery — dataset → mining → draft flow.
  • Users — policy maker, bureaucrat, consumer.
  • Flow runtime — config + deterministic harness + interaction layer.
  • Core services — audit / record store, case / identity store.
  • External services — registry, eID, payment, notification.
  • Fanar — model backend for harness and hotline.

Agentic Workflow Design

  • Planning / decomposition — the flow: nodes and edges.
  • Tool orchestration — per-tool allow / ask / deny gating.
  • Memory & state — case / identity store.
  • Retrieval — extract policy + database into the config.
  • Autonomous execution — harness runs the flow deterministically.
  • Multi-agent — subagents per task (own model / prompt / tools).
