01
The problem

Why aren't agents in government?

  • We rely on others to be correct, and accountable when wrong.
  • Agents are unreliable on some tasks.
  • and inconsistent, different answers, same input.
Flowstate's answer

Automate the routine, escalate the exceptions, stay accountable.

Why aren't LLM agents inside government work? Reliability. We rely on people to be correct and accountable when wrong. Agents are both unreliable on some tasks and inconsistent. Flowstate's answer: automate the routine deterministically, escalate only the real exceptions to a human, keep it accountable.

Flowstate is agentic by construction. Task decomposition: a whole procedure becomes typed nodes. The harness runs the flow autonomously end to end. Action nodes do tool orchestration — calling code and services under allow/ask/deny gating. Agent nodes are where Fanar does the planning and reasoning. Flow variables give memory and state across the run. And three flows — draft, run, update — cooperate as multiple agents. Every node tagged here is one of the agentic capabilities the rubric asks for.

Leveraging Fanar

Where Fanar fits

  • Agent backend, nodes that reason, classify, call tools.
  • Document parsing, text, images, audio.
  • Strong on Arabic context, a Gulf-gov fit.
One config · model family

chat vision speech

online (Fanar API) ⇄ self-hosted

Fanar plays two roles: the agentic LLM behind tool-calling nodes, and a document parser across text, images, and audio. Its Arabic strength fits a Gulf-government setting. Reached through one OpenAI-compatible config entry, online or self-hosted, swapped without touching code.

Fanar on one self-hosted instance, on par for Arabic extraction, just under the frontier on hard reasoning. The loop also mined 5 data-backed flow improvements from the 3% appeal cases.

We pulled procedure datasets to test effectiveness: a road-traffic fine event log and an Arabic legal-judgement set. Scoring is by script against a held-out key, the model never grades itself. Fanar on one self-hosted instance is on par with frontier models for Arabic extraction and conformance, slightly under on the hardest reasoning. And the update flow mined five concrete, data-backed improvements from the 3% of appealed cases.
