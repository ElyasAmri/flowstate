Flowstate

Codify bureaucratic procedures as deterministic workflows, with AI agents only where judgement is needed.

Smart Government & Citizen Services, powered by Fanar

Most procedures run automatically; humans stay in the loop for genuine exceptions. Team: Osama, Elyas.

The Problem

  • Routine applications are slow, inconsistent, and expensive when every case waits on a person.
  • Most cases are unambiguous; only a few are true exceptions.
  • Decisions must stay accountable: replayable and appealable.
  • Built for the Arabic / Gulf government context: dialect, local procedures.
Automate the routine, escalate the exceptions, keep it accountable.

How it works

  • A sealed, deterministic engine reaches the world only across typed channels.
  • Four node kinds: channel, agent (Fanar), action, decision.
  • The routine path runs deterministically with no model discretion.
  • Exactly one non-deterministic node, the agent; genuine exceptions pause at a human gate.
Every external/non-deterministic interaction is named, typed, visible on the canvas, hence auditable.

The loop

  • Draft: mine a routine flow from existing process data.
  • Run: routine cases auto-process; exceptions go to a human gate.
  • Improve: accumulated exceptions propose updates back to the flow.
  • All three are authored as real flows that compile to maestro YAML.
flow-drafting, fine-management-routine, flow-update in examples/. They close the loop.

Use of Fanar

Main

  • Powering the harness: the agent nodes that classify and decide.
  • Document parsing: text, images, transcript summaries.

Extra

  • Hotline as a separate flow: Arabic speech / dialect, integrated as a channel.
One config entry; online (Fanar API) or self-hosted, swapped without code changes.

Evaluation

  • Two public datasets: Road-Traffic Fines (150,370 cases) and Arabic-LJP (Saudi court).
  • Routine vs. exception split: 97% / 3%; classification 100% on 60 blind cases.
  • Arabic legal judgement: 86% on 50 blind cases; jurisdiction routing 100% precision.
  • The three loop flows compile clean (0 errors) to valid maestro YAML.
Claude stood in for Fanar (config-only swap). Scoring is by script vs. a held-out key. See eval/ and demo/run_demo.sh.
