The problem

Why aren't agents running institutions?

  • Unreliable on the tasks we hand them.
  • Inconsistent: same input, two different answers.
  • Unaccountable: no one answerable for the step.
The reliability framing (script): institutions rely on work being done correctly and someone being held accountable. Agents are not only fallible, they are inconsistent on the same input, so you can't drop one into the middle of a procedure.

The root cause

One agent, too many responsibilities.

Most work is a fixed order of steps. Split the concerns; coordinate the agents.

Too many tools, too many tasks, expecting wonders. The thesis: separation of concerns over the agents buys consistent, reliable output. Pause here.

How Flowstate works

An agent-driven program, inspired by Claude Code and n8n. Discovery mines a draft flow from datasets + policy. Three user roles: policy maker authors, bureaucrat signs off, consumer is served. The sealed harness runs the flow; external and core services are reached only across channels. Fanar is the model backend, in two roles: agentic tool-calling and document parsing (text / images / audio), and it reads Arabic best.
A case flowing through the editor: intake at the channel door, validate, the Fanar agent classifies, then either an automatic issue or an escalation to the human gate.

Eval design

Is the model deciding, or reading the answer?

  • Two layers are extraction: the answer is already in the input.
  • One is real judgement: we strip the court's reasoning, so it decides blind.
  • Stripping it drops every model from 100% to 56-84%.

What we learned

Models fail in opposite directions (over-reject vs over-grant). Jurisdiction is the universal hard tail, exactly the human-gate cases.

Conformance (routine vs exception) 100%, a deterministic guard. Arabic structured verdict 98-100%. Facts-only judgement 56-84%, the honest test. Bias direction sets the gate threshold; jurisdiction can't be inferred from raw facts.

Refinement

Flows improve themselves over time.

  • Unrecognized patterns feed an update flow.
  • It proposes a flow change; a human approves it.
  • Inputs are channels, so it can also pull a policy document or a ticketing system.

Why this shape wins

Deterministic

The routine path is guards. Same input, same path, every time.

Auditable

Every step leaves a typed, replayable trace.

Appealable

A citizen sees why; a human gate can review it.

One non-deterministic node (the agent); everything else is pure logic. Replayability plus a typed audit trail is what makes a decision appealable. Humans intervene only at gates.

Governance

Legal, private, secure by construction.

Legality

Require human signoff on every procedure where the law demands it.

Privacy

Self-host Fanar locally; case data never leaves the institution.

Security

Drop a filter (a QCRI service like AiXamine) as a gate between inputs and agents.

Legality: if signoff is legally required, every procedure can be signed off after processing. Privacy: running Fanar locally limits the concern. Security: the flexibility lets you put a filter like AiXamine between inputs and agents to screen malicious documents.
